# Onboarding Checklist

New hires follow a two-week onboarding sequence with shared responsibilities between the new hire and their manager. Week 1 focuses on access, training, and first contributions; Week 2 focuses on pairing, team integration, and core company rituals.

## Ownership

Managers own tasks assigned in the manager column. New hires own tasks assigned in the individual checklist.

## Week 1

### Day 1

New hire:
- Accept laptop
- Activate SSO
- Set up 1Password

Manager:
- Post an intro message in `#new-hires`

### Day 2

New hire:
- Read the [[employee-handbook]]
- Complete security training

Manager:
- Schedule 1:1s for weeks 2–4

### Day 3

New hire:
- Read the team's runbooks
- Set up the local development environment

Manager:
- Assign the first ramp-up task

### Day 5

New hire:
- Submit a first PR; a documentation fix is acceptable

Manager:
- Review the PR the same day

## Week 2

- Pair with a teammate on a real feature
- Attend the all-hands
- Attend the Friday wiki lint review
- Complete the [[context-engineering-primer]] reading
- Meet the [[onboarding-buddy]] for lunch
