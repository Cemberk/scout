# Onboarding checklist

The onboarding checklist covers a new hire's first two weeks at ACME. Managers own the manager-assigned items, and new hires own the individual setup and learning tasks.

## Week 1

### Day 1
New hires:
- Accept laptop
- Activate SSO
- Set up 1Password

Manager:
- Post an intro message in `#new-hires`

### Day 2
New hires:
- Read the [[employee-handbook]]
- Complete security training

Manager:
- Schedule 1:1s for weeks 2–4

### Day 3
New hires:
- Read the team's [[runbooks]]
- Set up the local development environment

Manager:
- Assign the first ramp-up task

### Day 5
New hires:
- Submit a first PR; a documentation fix is acceptable

Manager:
- Review the PR the same day

## Week 2

During week 2, the new hire:
- Pairs with a teammate on a real feature
- Attends the all-hands
- Attends the Friday wiki lint review
- Completes the context-engineering primer reading
- Meets the [[onboarding-buddy]] for lunch
