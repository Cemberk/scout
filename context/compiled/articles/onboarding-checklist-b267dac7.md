# Onboarding checklist

This checklist covers a new hire's first two weeks. Managers own manager tasks; new hires own the rest.

## First week

- **Day 1**
  - New hire: accept laptop, activate SSO, set up 1Password.
  - Manager: post an intro message in `#new-hires`.
- **Day 2**
  - New hire: read the `[[employee-handbook]]`, complete security training.
  - Manager: schedule 1:1s for weeks 2-4.
- **Day 3**
  - New hire: read team runbooks, set up local dev environment.
  - Manager: assign the first ramp-up task.
- **Day 5**
  - New hire: submit a first PR; a docs fix is acceptable.
  - Manager: review the PR the same day.

## Second week

- Pair with a teammate on a real feature.
- Attend all-hands and the Friday `[[wiki-lint-review]]`.
- Complete the context-engineering primer reading.
- Meet the `[[onboarding-buddy]]` for lunch.
