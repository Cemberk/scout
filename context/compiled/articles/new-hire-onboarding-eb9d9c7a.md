# New Hire Onboarding

The first two weeks of onboarding are split between new-hire tasks and manager tasks. New hires complete account setup, required reading, training, and an initial code contribution; managers handle introductions, scheduling, and the first ramp-up assignment.

## Ownership

Managers own items listed in the manager track. New hires own items listed in the individual track.

## Week 1

### Day 1

New hires:
- Accept laptop
- Activate SSO
- Set up 1Password

Managers:
- Post an intro message in `#new-hires`

### Day 2

New hires:
- Read the [[employee-handbook]]
- Complete [[security-training]]

Managers:
- Schedule 1:1s for weeks 2 through 4

### Day 3

New hires:
- Read the team's runbooks
- Set up the local development environment in [[developer-environment-setup]]

Managers:
- Assign the first ramp-up task

### Day 5

New hires:
- Submit a first pull request; a documentation fix is acceptable

Managers:
- Review the pull request the same day

## Week 2

Week 2 focuses on integration with the team and exposure to normal operating routines.

New hires:
- Pair with a teammate on a real feature
- Attend the all-hands
- Attend the Friday wiki lint review
- Complete the context-engineering primer reading
- Meet the [[onboarding-buddy]] for lunch
