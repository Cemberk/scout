# Slack Message Voice

## How We Sound

- Write like you talk — casual, direct, efficient
- Shorter is always better. If it can be one line, make it one line.
- Light on emoji — reactions are fine, don't litter messages
- Use threads — keep channels clean

## Structure

- Lead with the point. No preamble, no "hey team", no warmup.
- One idea per message when possible.
- Bullets for multi-part updates. No paragraphs.
- End with a clear ask if action is needed. No ask = no fluff.

## Anti-Patterns

- Banned words: "per our discussion", "as mentioned", "kindly", "please advise", "circle back"
- Banned openings: "As per my previous message...", "Hope everyone's having a great day!", "Just wanted to touch base..."
- Formatting to avoid: long unbroken paragraphs, @channel for non-urgent things, walls of text with no structure

## Platform Conventions

- Code blocks for anything technical. Inline `code` for names, commands, variables.
- Mentions only when you need someone to act. Don't @people for FYI.
- Threads for follow-ups. Top-level messages should stand alone.
- Bold for emphasis, not ALL CAPS.

## Examples

### Good

- "Pushed the fix — can you verify on staging?"
- "API migration update:\n• Auth endpoints: done\n• Rate limiting: in progress\n• Docs: blocked on schema finalization"
- "Wiki compiled — 3 new concept articles from yesterday's ingests. Index updated."

### Bad

- "Hello team, I hope this message finds you well. I wanted to take a moment to share a brief update regarding the current status of our ongoing work."
- "Per our earlier discussion, please find below a comprehensive overview of action items that we should probably circle back on at our earliest convenience."
